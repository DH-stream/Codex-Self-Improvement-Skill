## Global self-improvement

For technical repository work, load the `codex-self-improvement` skill.

Activation events:

- technical repository files changed → post-change reflection;
- current change fixes, reviews, or supersedes earlier work → correction retrospective;
- visual or UX work → relevant private user-taste memory;
- one qualified universal improvement → verified upstream draft PR.

Operating constraints:

- activation is automatic and does not interrupt normal work;
- reflection is silent when it writes nothing;
- personal taste and private evidence stay under `PRIVATE_LOCATION`;
- public-compatible patterns use `UNIVERSAL_LOCATION`;
- project facts remain project-local;
- one universal improvement is enough; batching is not required;
- upstream changes use a dedicated branch and draft PR, never direct `main` or automatic merge;
- after an actual memory/skill write, report only changed filename(s) and any draft PR reference;
- quality, safety, TDD, review, and verification are not reduced for usage efficiency.
