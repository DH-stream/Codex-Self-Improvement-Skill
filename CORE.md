# Core invariants

Load this before modifying the self-improvement engine or its memory.

1. **User intent and correctness come first.**
2. **Fire-and-forget means automatic and quiet, not invisible risk.**
3. **Efficiency may reduce repetition, scope, and narration; never required evidence.**
4. **One global engine serves every repository.**
5. **Private user learning and public universal learning are separate storage domains.**
6. **Personal taste and private evidence never enter public branches, commits, issues, or PRs.**
7. **Project facts are context, not global learning.**
8. **Explicit durable feedback and corrections are evaluated even when no code changes.**
9. **Technical efficiency reflection still requires an actual technical change.**
10. **`UNIVERSAL_LOCATION` is a read-only snapshot; upstream branches are the only writable universal source.**
11. **One qualified universal improvement may open a draft PR; batching is not required.**
12. **Universal contributions use stable IDs, deterministic branches, isolated worktrees, and current remote `main`.**
13. **Retries reuse existing queue/branch/PR state and run at most once per session or natural consolidation point.**
14. **Direct pushes to `main`, automatic merge, approval, and ready-for-review transitions are forbidden.**
15. **Nothing learned is deleted. Lower relevance, mark superseded, or transition operational status while preserving history.**
16. **Original evidence is immutable; later assessments append.**
17. **Routine reflection must cost less than the waste it prevents.**
18. **No write means no notice; a real write produces one compact filename/PR notice.**
19. **Installer replacements are validated and staged before active files are changed.**
20. **Engine changes are tracked and pressure-tested; ordinary private learning updates private memory.**
21. **Never claim success without fresh verification appropriate to the task.**
22. **Before multi-task execution, review dependencies, file ownership, and shared state; run safe independent work in bounded parallel worktrees, keep coupled work sequential, and preserve integrated verification.**
